# Topology diagrams

This directory contains sanitized network topology diagrams for the nix-config
repository. The diagram is generated by [nix-topology](https://github.com/oddlama/nix-topology)
from the flake's host configuration and the operator-detailed topology module.

## Files

- `overview.svg` — sanitized public overview, committed to the repo and
  displayed in the root README. Shows host roles, inter-host trust flows, and
  generic connection types — no MACs, private IPs, or hostnames.

## Regeneration

```bash
just topology
```

Updates `docs/topology/overview.svg` from the current flake state. Commit the
result if topology inputs changed.

## What's deliberately omitted

Per the [public repository data policy](../security/public-repository.md):

- MAC addresses and private IP addresses are redacted.
- Router and NAS infra are shown as shapes without connection detail.
- Detailed operator diagrams (showing interface names, backup paths, VLANs)
  are built locally only with `just topology-operator-detailed`.

## Shape legend

| Shape | Meaning |
| ----- | ------- |
| Rounded rectangle | NixOS host (soyo, zbook) |
| Rectangle | External device (router, NAS) |
| Diamond | Logical or virtual entity |
| Solid arrow | Primary data flow |
| Dashed arrow | Trust / management flow |

## Input files

The topology is generated from:

- `modules/parts/topology.nix` — flake-parts module that configures
  nix-topology with host and network definitions.
- `hosts/<name>/topology.nix` — host-specific hardware and connection data.
